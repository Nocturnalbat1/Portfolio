Completed courses

B.S. Cyber Intelligence & Security — highlights from core + concentration classes.

Core (Cyber Intelligence & Security)

CI 120 Intro to Cyber Security Majors

This course provides students with a comprehensive overview of cyber security with extensive hands on exercises to demonstrate the concepts. An academic foundation will provide the students with the vocabulary and concepts to understand internet, basic computer concepts, the relationships between data, applications, systems and hosts. Laboratory exercises will allow the students to explore and extend their interactive lectures to discover experientially the concepts that are presented in lecture.

CS 125 Computer Science I

Introduction to problem-solving methods, algorithm development, and software engineering; software development process, program design, coding, review, testing, and documentation; and programming using a modern programming language that supports modular development.

CS 213 Intro to Computer Networks

This course built a strong foundation in how data moves across networks, from basic packet flow to protocol-level communication. I learned how IP addressing, routing, switching, and transport-layer protocols work together to enable reliable communication. Understanding these mechanics was critical for recognizing how misconfigurations, weak segmentation, or insecure protocols can introduce vulnerabilities. This course directly influenced my interest in network security, telemetry analysis, and later work involving traffic inspection and threat detection.

CS 315 Data Structures / Algorithms

Explored efficient ways to store, organize, and process data using structures such as lists, trees, and graphs. Strengthened logical thinking and performance awareness when designing software solutions.

CS 420 Operating Systems

Operating Systems went deep into how modern systems manage hardware and software resources. I studied process scheduling, memory management, file systems, and kernel-level behavior, gaining insight into how software interacts with hardware at a low level. This class helped me understand where attackers can hide, how privilege escalation occurs, and why OS-level visibility is essential for detection and defense. It also strengthened my ability to reason about performance, isolation, and system stability—skills that carry over into malware analysis and embedded systems security.

CI 330 Software Security

This course focused on how software design decisions can introduce security risks long before code is deployed. I analyzed common vulnerabilities such as input validation flaws, insecure authentication, and poor error handling. Emphasis was placed on thinking like both a developer and an attacker—understanding how insecure logic can be exploited and how to prevent it through secure coding practices. This class reinforced the idea that security must be built in, not bolted on.

CI 340 Database System Security

Studied database architectures and security controls used to protect data at rest and in transit. Covered access control, authentication, and common database attack vectors.

Offense & Defense

CI 320 Ethical Hacking / Pen Testing

This course introduced me to offensive security from a structured and ethical perspective, focusing on how real attackers identify and exploit weaknesses in systems and networks. I learned how reconnaissance, enumeration, and misconfigurations often serve as the starting point for larger compromises, and how small flaws can be chained together to gain deeper access. Rather than simply running tools, the course emphasized understanding attacker methodology, intent, and decision-making. This mindset helped me think critically about how systems fail under real conditions and reinforced the importance of designing defenses that account for how attackers actually operate.

CI 311 Securing Computer Networks

Securing Computer Networks focused on building and defending networks with security as a core design principle rather than an afterthought. The course explored how firewalls, intrusion detection systems, access controls, and network segmentation work together to protect enterprise environments. I learned how attackers move laterally once inside a network and how proper architecture, monitoring, and visibility can limit that movement or expose malicious activity early. This course strengthened my understanding of network-level telemetry, traffic analysis, and the role of layered defenses in protecting complex systems.

CS 432 Info & Computer Security

This course provided a broader systems-level view of security by connecting technical controls with risk management and real-world constraints. I learned how confidentiality, integrity, and availability are balanced in practical environments and why security decisions often involve trade-offs between usability, performance, and protection. The course emphasized threat modeling, policy considerations, and how human behavior and organizational structure influence security outcomes. This helped me think beyond individual vulnerabilities and evaluate security posture at a system and organizational level.

SIS 210 Security Fundamentals

Security Fundamentals introduced the core concepts that support both offensive and defensive security practices. The course emphasized understanding threats, vulnerabilities, and risk across physical, digital, and organizational domains. It helped establish a common security mindset and vocabulary, allowing me to better connect technical controls with real-world security challenges. This foundation supported later coursework by framing cybersecurity as a discipline that spans technology, policy, and human factors.

CS 303 Cryptography & Network Security

Cryptography & Network Security focused on the foundations of secure communication and trust in modern systems. The course explored how encryption, hashing, authentication, and key management are used to protect data and communications, as well as how improper implementation can completely undermine security. I gained a deeper understanding of where cryptography fits within real networks and how protocols rely on cryptographic primitives to ensure confidentiality and integrity. This course reinforced the importance of correct implementation and helped me better understand secure communications in networked and embedded environments

Forensics & Investigation

CS 304 Intro to Computer Forensics

This course introduced the fundamentals of digital forensics and how cyber incidents are investigated from an evidentiary standpoint. I learned how digital evidence is identified, preserved, and analyzed while maintaining proper forensic integrity. The course emphasized understanding file systems, system artifacts, and user activity to reconstruct events accurately. It strengthened my ability to think methodically, follow investigative procedures, and translate technical findings into clear conclusions that can support incident response or legal processes.

CI 450 Computer Forensics II

Building on introductory forensic concepts, this course moved into more advanced investigative scenarios and deeper system analysis. I worked with more complex artifacts and learned how to correlate data across multiple sources to reconstruct timelines and attacker activity. The emphasis on documentation, accuracy, and analytical reasoning reinforced the importance of precision during investigations. This course strengthened my ability to approach compromised systems with a structured mindset and extract meaningful insights from large amounts of data.

CI 410 Malware Analysis

Malware Analysis focused on understanding malicious software by studying its behavior, structure, and intent. I learned how malware executes, persists, communicates, and evades detection, and how those behaviors can be identified and analyzed. Rather than focusing solely on tools, the course emphasized understanding patterns and attacker objectives. This strengthened my analytical thinking and helped me connect malware behavior to defensive strategies, detection engineering, and real-world threat response.

Intelligence Concentration

CI 310 Intel, Surveillance & Recon

This course explored how intelligence is gathered, processed, and used to support decision-making in security operations. I learned how surveillance and reconnaissance techniques contribute to situational awareness and how raw data becomes actionable intelligence. The course emphasized analytical thinking, context, and ethical considerations when working with intelligence sources. It helped me better understand how technical data collection ties into strategic analysis and operational planning.

SIS 315 Studies in Global Intelligence I

Studies in Global Intelligence examined how intelligence organizations operate within a global and geopolitical context. The course focused on intelligence structures, real-world case studies, and how intelligence influences national security decisions. This class helped me understand how cybersecurity fits into larger intelligence and defense frameworks, reinforcing the idea that technical security decisions often have broader political and strategic implications.

Supporting Courses

CI 201 Intro Linux, Unix, Windows, Scripting

This course provided hands-on experience working across multiple operating systems and environments. I developed practical skills using command-line interfaces, writing scripts, and navigating system internals. Understanding how Linux, Unix, and Windows differ strengthened my ability to troubleshoot systems, automate tasks, and analyze behavior from a security perspective. These skills are foundational for both offensive and defensive cybersecurity work.

ACC 210 Financial Accounting

The use of financial accounting information in business operations. The accounting cycle; adjusting entries; merchandising operations; inventory; depreciation; accounting for assets, liabilities, and stockholder's equity; financial statement analysis.

ACC 329 Forensic Accounting Fraud Exam

This course focused on identifying and investigating financial fraud through analytical and investigative techniques. I learned how patterns, anomalies, and red flags can indicate fraudulent activity and how evidence is gathered to support findings. The course strengthened my investigative mindset and helped me understand how cybercrime, fraud, and financial misconduct often overlap, especially in complex organizational environments.

MA 225 Intro to Discrete Structures

This course focused on the mathematical foundations that support computer science and cybersecurity systems. I studied logical reasoning, sets, relations, functions, graph theory, and proof techniques, all of which are critical for understanding how algorithms and secure systems are designed. Discrete Structures strengthened my problem-solving mindset by teaching me how to break complex problems into structured, logical steps. The concepts learned in this course directly support areas like cryptography, network design, data structures, and formal security reasoning, helping me think more rigorously about how systems behave and how vulnerabilities can emerge from flawed logic.

CI 460 Data Analysis and Machine Learning

This course explored how large datasets are processed, analyzed, and transformed into actionable insights using machine learning techniques. I learned how data is collected, cleaned, and structured before being used in analytical models, as well as how machine learning algorithms identify patterns, trends, and anomalies. The course emphasized practical applications of data analytics and AI, connecting statistical methods with real-world decision-making. From a cybersecurity perspective, this class strengthened my understanding of how machine learning can be applied to threat detection, anomaly detection, and predictive security models, especially in environments with high-volume telemetry and limited resources.
